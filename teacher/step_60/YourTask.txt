Ментор обратил ваше внимание на то, как система возвращает ошибки
(класс ValidationError). По его мнению привязывать ошибки к полю
в запросе (private String field) не совсем корректно. Например, могут
быть ошибки связанные сразу с несколькими полями запроса.

Ментор предлагает поменять в классе ValidationError

private String field;
на
private String errorCode;

а так же
private String message;
на
private String description;

Далее ментор предлагает создать в папке main/resources
файл errorCodes.properties и разместить в этом файле список
всех кодов ошибок и соответствующих им описаний:
ERROR_CODE_1=Field personFirstName is empty!
ERROR_CODE_2=Field personLastName is empty!
ERROR_CODE_3=Field agreementDateFrom is empty!
и так далее.

Таким образом файл errorCodes.properties будет содержать список
всех кодов ошибок и их описаний.

Далее ментор предлагает убрать из кода приложения все описания
ошибок и доставать их во время работы приложения из файла
errorCodes.properties по коду ошибки. Для этого нужно разобраться
как в Spring можно загрузить файл с параметрами во время запуска
приложения в память и как потом из него читать значения параметров.
Написать компонент/класс который позволит по коду ошибки получать
ее описание из файла errorCodes.properties во время выполнения программы
и внедрить его в приложение.

После указанных изменений все коды ошибок и их описания будут лежать
в одном месте, в файле errorCodes.properties. Это позволит поддерживать
уникальный набор кодов ошибок в одном месте приложения. В коде приложения
останутся только коды ошибок.


План реализации предложенных изменений:
1. Поменять названия полей в классе ValidationError
field -> errorCode
message -> description

2. Исправить названия полей ошибок во всех файлах так,
чтобы все тесты проходили.

3. Создать файл errorCodes.properties.

4. Найти в интернете способ, как во время выполнения приложения в Spring
доставать из файла с ресурсами значения свойств.

5. Реализовать найденный из предыдущего пункта способ так, чтобы
по коду ошибки можно было получить ее описание из errorCodes.properties.

6. Пройтись по всем местам, где приложение создаёт ошибки.
Каждой ошибке присвоить уникальный код (ERROR_CODE_1, ERROR_CODE_2, ...)
и добавить ее в файл errorCodes.properties вместе с описанием. Если
необходимо, то подкорректировать описание ошибки. Убрать из кода описание
ошибки и внедрить код по получению описания ошибки по коду ошибки.

7. Убедиться, что все тесты корректно работают.
