Ментор обращает ваше внимание, что текущая реализация джоба
по выгрузке агриментов в xml файлы не оптимальна. Она происходит
в одном потоке и если агриментов в базе будет очень много, то
выгрузка будет выполняться очень долго.

Ментор просит вас ускорить выгрузку агриментов в xml файлы
используя потоки и предлагает воспользоваться классом
ThreadPoolExecutor.

Сценарий работы приложения должен быть таким:
1. Запускается джоб по выгрузке агриментов в xml файлы;
2. Джоб запрашивает и получает все агремент uuids с помощью команды;
3. Джоб создает ExecutorService с указанным в application.properties
   количеством потоков;
4. Джоб для обработки каждого агримент uuid создает задачу и сабмитит
   ее в ExecutorService для выполнения;
5. Джоб дожидается выполнения всех задач и выключат ExecutorService
   и после этого завершает свою работу.

Ваша задача: реализовать предложение ментора.

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Step_X_description.txt. Формат заполнения этого
файла описан в step_1 в файле Learning_reflection.txt.
