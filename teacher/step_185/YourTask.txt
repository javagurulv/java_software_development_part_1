Ментор обращаем ваше внимание на план реализации
находящийся в папке documentation в предыдущем шаге.

В этом плане есть некоторые моменты, которые нужно исправить:

Запомните одну вещь. Перед реализацией любого алгоритма
вы должны четко понимать:
- с какими данными будет иметь дело ваш алгоритм;
- сколько их будет;
- как они будут представлены (их формат);
- как будет происходить их обработка.
От ответов на эти вопросы будет многое зависеть!

1. Загружать все агрименты в память приложения нельзя.
При больших объемах данных в базе данных загрузка их в память
приложения просто не возможна. Всегда помните, что в базе данных
может оказаться десятки и сотни миллионов записей. Вы разрабатываете
код не для тестовой среды, а для реального продакшн.

2. Какие названия будут у файлов?

3. Что делать в том случае, если в директории, где создаются
файлы уже есть файлы с такими названиями? Функционал должен
отрабатывать корректно при повторном выполнении.

4. Для новой функциональности нужна возможность ее включать
и выключать.

Плюс ментор хочет, чтобы вы рассмотрели и архитектурную часть задания.
С точки зрения приложения джоб по выгрузке агриментов в xml файлы
можно рассматривать как внешнюю интеграцию (такую же как web и REST).
То есть ее можно вынести из ядра (core) приложения и реализовать так,
чтобы она получала данные из ядра используя внешний API core.
Для такой реализации потребуется доработать некоторые команды.

Ваша задача: доработать план учтя все замечания приведенные выше.
