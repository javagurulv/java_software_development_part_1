От бизнеса поступила следующая информация.
Бизнес хочет начать масштабную рекламную компанию
по привлечению новых клиентов в наше приложение.
Бизнес просит нас убедиться в том, что наше приложение
справится с большим числом новых пользователей.

Большое количество новых пользователей приведёт к возрастанию
количества запросов к приложению. Понятное дело, что бесконечно
увеличивать число обрабатываемых запросов мы не сможем. Нам надо
выяснить, с какой нагрузкой сможет справиться наше приложение.

Для этого нам надо провести так называемый Load Testing (нагрузочное
тестирование). Цель этого способа тестирования выяснить с каким
количеством запросов в единицу времени способно справляться
приложение.

Для проведения нагрузочного тестирования в реальной жизни используются
платные приложения. Так как мы на данном этапе не можем себе этого
позволить, нам нужно разработать собственный код для нагрузочного
тестирования.

Ваша задача:
В Spring библиотеке есть класс под названием RestTemplate.
С помощью данного класса можно выполнять REST запросы.

Создайте в тест директории пакет loadtesting. В этом пакете
создайте класс RestCallExample с main(...) методом, чтобы этот
класс можно было запустить как маленькое приложение.

Внутри main(...) метода напишите код, который при помощи RestTemplate
делает два запроса (/v1 и /v2) к приложению поднятому на localhost.
Json запросов и ответов возьмите из ресурсной директории тестов.

Код должен:
- прочитать json запроса из файла в строку;
- прочитать json ответа из файла в строку;
- выполнить REST запрос к приложению при помощи класса RestTemplate;
- получить тело ответа как строку;
- сравнить полученный ответ от сервера и
  заранее прочитанный ответ из файла.

Для чтения запросов и ответов из файлов
и сравнения ответов воспользуйтесь кодом из тестов.

После написания кода запустите приложение и сделайте к нему запросы.

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Step_X_description.txt. Формат заполнения этого
файла описан в step_1 в файле Learning_reflection.txt.
