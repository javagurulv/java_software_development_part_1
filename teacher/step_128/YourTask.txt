У клиента возникла проблема с установкой программы на прод среду.
Проблема заключается в том, что перед запуском программы нужно
создать базу данных и запустить скрипты для создания структуры
и заполнения таблиц начальными значениями. Клиент создал
базу данных, но забыл выполнить SQL скрипты. При запуске приложение
упало с ошибкой.

Клиент интересуется, можно ли сделать так, чтобы скрипты по
созданию структуры базы данных и заполнению таблиц начальными
значениями выполнялись приложением автоматически при запуске,
чтобы не надо было их отдельно руками запускать.

Ментор добавил в приложение поддержку Liquibase.
Изменения коснулись следующих файлов:
/build.gradle
/src/main/resources/application.properties
/src/test/resources/application.properties
/src/main/resources/db/changelog/*

Ваша задача:
Познакомиться с Liquibase.
Разобраться с изменениями сделанными менторов в приложении.
Провести эксперимент: удалить базу данных, создать пустую базу
данных с таким же названием, без создания структуры таблиц
запустить приложение, изучить что произойдёт с базой данных,
объяснить как это произошло и почему. Выключить приложение,
включить его повторно и убедиться, что оно не упадёт и для таблиц
в базе данных не будут созданы дубликаты, объяснить почему.

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Step_X_description.txt. Формат заполнения этого
файла описан в step_1 в файле Learning_reflection.txt.
