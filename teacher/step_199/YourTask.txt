Ваша задача:
Создайте класс LoadTestingStatistic в пакете loadtesting.
Этот класс будет предназначен для сбора статистики по
времени выполнения запросов при нагрузочном тестировании.

В этом классе должен быть список:
    private List<Long> executionTimes = new ArrayList<>();
и методы для добавления значений в список,
методы расчёта среднего/максимального/минимального значения.

Все методы должны быть synchronized так как этот класс
будет применяться в многопоточной среде.

Добавьте конструкторы в классы, которые реализуют
интерфейс Runnable, для того чтобы в них можно было передать
экземпляр класса LoadTestingStatistic, сохраните его в свойство
класса, и в конце выполнения запроса добавляйте в него высчитанное
значение времени выполнения запроса.

В начале программы в классе RestCallExample создайте
по одной инстанции класса LoadTestingStatistic для каждой
версии /v1 и /v2 и подайте их в конструкторы.

Сохраните все созданные потоки в список:
        List<Thread> threads = new ArrayList<>();

Добавьте в конце main(...) метода следующий код:

        threads.forEach(thread -> {
            try {
                thread.join(); // заставляет поток main() метода дождаться завершения работы потока
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        });

В конце main(...) метода (после join) добавьте вывод на экран
среднего арифметического, минимального и максимального времени
выполнения запросов для каждой версии v1 и v2.

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Step_X_description.txt. Формат заполнения этого
файла описан в step_1 в файле Learning_reflection.txt.
