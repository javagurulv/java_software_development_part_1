В базе данных приложения на продакшене начали сохраняться
очень ценные для бизнеса данные по клиентам и страховым полюсам.
Бизнес хочет открыть доступ к этим данным другим приложениям
компании, таким как корпоративная система расчёта рисков. Эта
система рассчитывает всевозможные риски связанные с бизнесом
компании и на их основе строит прогностические модели. Для расчёта
рисков и построения как можно более точных прогностических моделей
этой системе нужно как можно больше данных о клиентах и бизнес
продуктах компании (такие, как наши полюса страхования).

Бизнес ставит перед нами задачу - в кратчайшие сроки разработать
внутренний REST API, которым сможет воспользоваться корпоративная
система расчёта рисков для доступа к данным нашего приложения.

Требования к внутреннему REST API:
- получение полной информации по полису страхования используя
  уникальный идентификатор полиса;
- поиск клиентов по имени, фамилии и персональному коду;
- возможность запросить данные по всем полисам клиента.


Ментор предлагает начать с разработки REST вызова для получения
полной информации по полису страхования используя уникальный
идентификатор полиса.

Ваша задача: разработайте детальный план реализации данной задачи.
После составления плана, обсудите его с вашим ментором.
