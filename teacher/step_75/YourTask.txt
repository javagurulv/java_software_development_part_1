Шагом ранее мы внедрили архитектурные изменения позволяющие
рассчитывать премию для каждого риска в отдельности, а потом
суммировать премии рисков, чтобы получить общую премию полиса.

Так как приложение считает премию каждого выбранного риска в
отдельности, то у ментора возник резонный вопрос: "А нужно ли
отображать в ответе премию каждого выбранного риска?"

Это отличный вопрос от разработчика! Такие вопросы улучшают
коммуникацию между ИТ и бизнесом и позволяют создавать качественное
программное обеспечение. ИТ тоже может и должно предлагать идеи
по улучшению разрабатываемого программного обеспечения.

Задав вопрос выше бизнесу ментор получил положительный ответ.
Бизнес хочет видеть в ответе премии для каждого выбранного риска
в отдельности и общую премию для всего полиса.

Ментор предлагает следующий формат ответа:

{
  "errors": null,
  "personFirstName": "Vasja",
  "personLastName": "Pupkin",
  "agreementDateFrom": "2025-05-25",
  "agreementDateTo": "2025-05-29",
  "agreementPremium": 4,
  "risks": [
          {
            "riskIc": "TRAVEL_MEDICAL",
            "premium": 2.00
          },
          {
             "riskIc": "TRAVEL_CANCELLATION",
             "premium": 5.00
          },
  ]
}


Ваша задача:

1. Добавить в ответ секцию "risks" как описано выше
(подсказка, секцию "risks" можно реализовать по аналогии с "errors").
На данном этапе заполнять секцию "risks" реальными данными не надо
(это потребует изменений в архитектуре расчёта премии, ментор
предлагает сделать это отдельно следующим шагом). Для проверки
правильности формата ответа на данном этапе заполните секцию "risks"
