Ментор выполнил первые шаги плана по реализации второй версии приложения
для смены направления зависимости:  /core <- /rest/v1 and /web/v1:

1. Создать отдельный модуль для реализации второй версии приложения.
Идея заключается в том, чтобы создать постепенно новый вариант
решения и потом удалить старый. Изменить старый постепенно не получится,
там надо менять всё сразу - а это слишком большие и не контролируемые
изменения в коде.

2. В новом модуле создать пакет /core/api. В этом пакете реализовать
новый API для core, от которого будут зависеть в будущем REST и WEB.


Ваша задача: изучить структуру классов и их код в новом модуле.
При необходимости уточнить все возникшие вопросы у ментора.
