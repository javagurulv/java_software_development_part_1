Ментор указал вам на то, что в процессе запуска приложения
или выполнения тестов в проекте создаётся директория /logs.
В этой директории хранятся логи. И файлы с этой директории иногда
попадают в Git.

Ваша задача (техническое улучшение):
сделать так, чтобы при выполнении команды gradle clean
директория /logs удалялась. Тогда перед коммитом изменений
в Git можно будет выполнить команду gradle clean и все
не нужные для коммита файлы будут удалены.

Команду gradle clean можно запустить из консоли воспользовавшись
Gradle Wrapper и файлов gradlew (находится в корне проекта)
выполнив команду ./gradlew clean.

PS: для решения этой задачи вам нужно внести изменения в
gradle.build файл проекта. Найдите в интернете пример удаления
директории при выполнении команды gradle clean и сделайте по аналогии.

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Step_X_description.txt. Формат заполнения этого
файла описан в step_1 в файле Learning_reflection.txt.
